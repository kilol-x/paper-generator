\section{引言}
\subsection{编写目的}
本文档为在线论文文档自动生成工具详细设计说明书。本文档立足于项目负责人及基础架构工程师视角，旨在将概要设计阶段确定的“用户鉴权与角色控制”模块进行具体代码化与物理结构化设计。文档详细定义了核心数据库字段约束与索引、RESTful 请求与返回的 JSON 载荷、鉴权过滤算法伪代码及安全防御逻辑，作为后续基础架构编码以及全组合规接入的直接实现蓝本。

\subsection{项目背景}
本系统为 Web 端在线论文编辑与标准化文档生成系统，由实训开发小组协同开发。基础架构模块作为整个系统的底座，必须优先完成编码交付，确保模板管理、富文本编辑器、文档导出、线上批阅等业务功能模块能基于统一的用户上下文及权限网关进行并行开发。

\subsection{定义}
\begin{itemize}
    \item \textbf{BCrypt}：一种基于 Blowfish 加密算法的强哈希密码散列函数，内置随机加盐机制，具备自适应工作因子，能有效防御彩虹表与暴力破解攻击。
    \item \textbf{JWTInterceptor}：后端控制层拦截器，用于在执行具体控制器逻辑前，对 HTTP 请求头中的 Token 进行提取、验签、解码及权限校验。
    \item \textbf{Security Context}：系统运行期线程上下文，用于安全地暂存当前通过鉴权的用户实体信息，供同线程内的业务代码随时获取。
\end{itemize}

\section{任务概述}
\subsection{需求概述}
本系统需建立完善的统一用户中心及基于角色的权限控制体系（RBAC）。系统强制划分三类核心角色：系统管理员负责院系与多类型论文模板管控；学生用户负责选用模板、在线编辑、暂存草稿及导出文档；教师用户拥有课程论文的专属批阅、添加批注、评定成绩与退回权限。基础架构必须在技术底座上实现无状态鉴权与全方位的越权拦截。

\subsection{运行环境}
\begin{itemize}
    \item \textbf{客户端环境}：支持主流现代化 Web 浏览器（Chrome, Edge, Safari, Firefox 等）；
    \item \textbf{服务端环境}：Linux/Ubuntu 22.04 LTS，Java 17 或 Python 3.10 运行环境；
    \item \textbf{数据库环境}：MySQL 8.0 关系型数据库，Redis 7.0 缓存服务器。
\end{itemize}

\subsection{条件与限制}
由于本系统属于高校软件工程实训项目，开发周期紧凑。基础架构人员（人员 1）必须比其他业务模块开发人员提早 3--5 天完成统一接口规范定义、无状态拦截器骨架搭建以及底层数据库表物理建表，否则将直接阻塞后续富文本编辑与批阅模块的串联开发。


\section{模块详细设计}
\subsection{用户权限控制模块设计}

\subsubsection{模块概述}
本模块是系统的核心安全中枢，承担全站用户的注册登录、身份凭证分发（JWT 签发）、无状态请求拦截验签、以及基于 RBAC 模型的管理员（ADMIN）、学生（STUDENT）、教师（TEACHER）三类角色的权限阻断与放行。本模块为全站提供统一的、基于线程绑定的用户上下文（UserContext）。

\subsubsection{模块输入输出接口}
本模块面向前端提供统一的 RESTful 标准 JSON 接口，以登录验证接口为例：
\begin{enumerate}
    \item \textbf{用户登录认证接口}：
        \begin{itemize}
            \item \textbf{请求 URL}：\texttt{POST /api/auth/login}
            \item \textbf{请求 Payload (JSON)}：
\begin{verbatim}
{
  "username": "202611001",
  "password": "mypassword123",
  "role": "STUDENT"
}
\end{verbatim}
            \item \textbf{返回 Data (JSON)}：
\begin{verbatim}
{
  "code": 200,
  "message": "Login Success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expireTime": 1783932400,
    "userId": 10021,
    "realName": "张同学"
  }
}
\end{verbatim}
        \end{itemize}
\end{enumerate}

\subsubsection{模块处理逻辑流程}
用户访问系统时，基础架构鉴权拦截器的处理逻辑流程如下：
1. 前端发送请求 $\rightarrow$ 2. 拦截器判断是否为免鉴权白名单（如 \texttt{/api/auth/login}） $\rightarrow$ 3. 若是则直接放行；若否则从请求头提取 \texttt{Authorization} $\rightarrow$ 4. 解析并验证 JWT 签名与失效时间 $\rightarrow$ 5. 解析对应角色，匹配当前路由访问权限 $\rightarrow$ 6. 成功后写入运行期线程上下文（Security Context）并放行至后续业务控制器。

\subsubsection{模块内部数据结构}
后端运行期内存中维护的用户上下文实体（供全组随取随用）结构定义如下：
\begin{verbatim}
@dataclasses.dataclass
class UserContextEntity:
    user_id: int          # 用户唯一物理ID
    username: str         # 登录账号(学号/工号)
    role_code: str        # 角色编码: ADMIN / STUDENT / TEACHER
    college_id: int       # 所属学院ID
    client_ip: str        # 客户端请求IP
\end{verbatim}

\subsubsection{模块算法设计}
模块核心的“全局请求鉴权过滤算法”在后文“第6节 算法详细设计”中统一展开实现。

\subsubsection{模块异常处理逻辑}
\begin{itemize}
    \item 当请求未携带 Token、Token 格式错误或验签失败时，抛出 \texttt{UnauthenticatedException} 运行时异常，拦截器直接就地阻断。
    \item 当解析的角色与当前请求的路由权限边界不匹配（如学生企图调用管理员接口），抛出 \texttt{UnauthorizedException}，终止业务流转。
\end{itemize}

\section{算法详细设计}
\subsection{核心算法：全局请求鉴权与 RBAC 隔离算法}
\subsubsection{算法描述}
拦截器对进入系统的 HTTP 请求执行动态路由拆包、合法性验签、时效性检查，并严格依据路由前缀实施多角色垂直越权拦截，成功后将用户实体封装挂载至线程上下文。

\subsubsection{输入输出参数}
\begin{itemize}
    \item \textbf{输入}：\texttt{request} (HTTP 请求封装对象)，包含请求头及 RequestURI。
    \item \textbf{输出}：\texttt{boolean} (True 表示允许放行进入核心业务层，False 拒绝)。
\end{itemize}

\subsubsection{代码实现}
\begin{verbatim}
def pre_handle_intercept(request, response) -> bool:
    # 1. 路由白名单直接放行(如登录接口)
    uri = request.get_request_uri()
    if uri.startswith("/api/auth/"):
        return True
        
    # 2. 从HTTP Header中安全提取Token凭证
    auth_header = request.get_header("Authorization")
    if not auth_header or not auth_header.startswith("Bearer "):
        raise UnauthenticatedException("Missing identity credentials.")
    token = auth_header.substring(7)
    
    # 3. 解析并校验JWT合法性与时效
    payload = jwt_provider.validate_and_parse(token)
    if payload is None or payload.is_expired():
        raise UnauthenticatedException("Token has expired or is invalid.")
        
    # 4. 基于业务路由前缀执行严格的RBAC垂直越权拦截
    user_role = payload.role_code
    if uri.startswith("/api/admin/") and user_role != "ADMIN":
        raise UnauthorizedException("Admin operations restricted.")
    if uri.startswith("/api/student/") and user_role != "STUDENT":
        raise UnauthorizedException("Student operations restricted.")
    if uri.startswith("/api/teacher/") and user_role != "TEACHER":
        raise UnauthorizedException("Teacher operations restricted.")
        
    # 5. 组装全局线程上下文, 供同小组其他功能模块并行开发时一键获取当前操作人
    context_user = UserContextEntity(
        user_id=payload.user_id,
        username=payload.username,
        role_code=user_role,
        college_id=payload.college_id,
        client_ip=request.get_remote_ip()
    )
    SecurityContextHolder.set_context(context_user)
    return True
\end{verbatim}

\subsubsection{边界条件处理}
若高频客户端在极短时间内发起并发请求，且遭遇 Token 刚好过期的交界点，系统设计了双 Token 续期机制（Refresh Token），在边界状态下允许前端静默刷新，提升学生在线撰写论文时的用户体验。

\section{出错处理详细设计}
\subsection{模块内部异常分类}
包含 \texttt{UnauthenticatedException}（401 未登录认证）与 \texttt{UnauthorizedException}（403 无对应角色权限/越权访问）。
\subsection{异常捕获流程}
任何业务层抛出的安全异常，均向上抛出并最终由基础架构的全局异常处理器（Global Exception Handler）切面统一捕获。
\subsection{异常返回信息定义}
触发越权攻击时，系统统一熔断并返回：
\begin{verbatim}
{
  "code": 403,
  "message": "Access Denied: Illegal privilege escalation detected.",
  "data": null
}
\end{verbatim}


\section{安全详细设计}
\subsection{身份校验逻辑}
全面实行基于 JWT 的无状态感知转换，密钥密存。
\subsection{数据加密处理设计}
用户密码全部采用 BCrypt 哈希算法加盐存储，严禁明文暴露。
\subsection{防攻击处理逻辑（防水平越权）}
为了防范水平越权（如学生 A 修改学生 B 的论文），基础架构强制约束后续论文编辑模块的代码逻辑必须按如下规则设计：
\begin{verbatim}
# 基础架构硬性约束的防水平越权代码范式
current_uid = SecurityContextHolder.get_context().user_id
current_paper = paper_repository.query_by_id(request.get_paper_id())
if current_paper.creator_id != current_uid:
    raise UnauthorizedException("Horizontal privilege escalation: Permission denied.")
\end{verbatim}

包含 JWT 过期时间（默认 2 小时）、数据库物理连接池上限等核心参数，统一通过外部配置文件维护。
